## Motivation

As AI language models become increasingly fluent and accessible, the ability to reliably distinguish AI-generated text from human writing has become a practical necessity across education, journalism, academic publishing, and content moderation. Current commercial detectors suffer from high false-positive rates, lack of transparency, and poor generalization across AI models and writing styles. This project builds a full ML pipeline ŌĆö from feature engineering through transformer embeddings to a deployable Streamlit application ŌĆö that is interpretable, reproducible, and evaluated on text generated by a real LLM.

---

## Project Overview

This project progresses across two capstone courses, developing an AI vs. Human text classifier from first principles.

**Capstone 1** establishes a feature-driven classical ML pipeline: TF-IDF n-grams, stylometric feature engineering, and baseline Logistic Regression / SVM classifiers. Synonym Replacement (SR) augmentation produces a large percentage-point F1 gain over the raw baseline.

**Capstone 2** extends to model optimization (Random Forest, Gradient Boosting), deep error analysis, SHAP global explainability, LIME word-level explainability, frozen DistilBERT transformer embeddings, and evaluation on 80 genuinely GPT-4o-mini-generated text samples. The final pipeline is a hybrid BERT + stylometric classifier served via Streamlit.

**The key finding:** The classical TF-IDF Random Forest achieves strong in-distribution performance but fails completely on real LLM data, indicating overfitting to synthetic (Faker-generated) patterns rather than true AI detection signals.
In contrast, hybrid models using BERT + stylometric features generalize significantly better.

This is the core motivation for the transformer approach and the real LLM evaluation.

## Data Preprocessing

**Cleaning steps:**
- Lowercased all text; removed HTML tags, extraneous symbols, and normalized whitespace
- Missing values: 199 entries across `flesch_reading_ease`, `gunning_fog_index`, `passive_voice_ratio`, and `sentiment_score` ŌĆö imputed using column means (missingness was random, not systematic)

**Feature engineering:**
- **14 intrinsic features** ŌĆö carried forward from dataset
- **16 advanced linguistic features** ŌĆö extracted via TextBlob and custom functions: sentiment polarity/subjectivity, stopword ratio, uppercase/digit/special character ratios, long-word ratio, punctuation event counts (question marks, exclamations, commas, semicolons), Yule's root TTR
- **TF-IDF word n-grams** ŌĆö up to 2,000 features, `ngram_range=(1,3)`, `min_df=3`, `max_df=0.85`, `sublinear_tf=True`
- **Character n-grams** ŌĆö up to 500 features, `analyzer='char'`, `ngram_range=(2,4)` ŌĆö captures punctuation habits and sub-word stylometric patterns
- **SelectKBest (chi-squared)** ŌĆö reduced to top 800 most label-discriminative features
- **StandardScaler** ŌĆö applied before LR/SVM training
- **Synonym Replacement (SR) augmentation** ŌĆö doubled training corpus to 2,734 samples using WordNet synonyms; fuzzy deduplication at cosine similarity threshold 0.99 reduced to 2,510 samples
- **80/20 stratified train/test split** ŌĆö `random_state=42`

## Modeling Approach

### Baseline Models (Capstone 1 ŌĆö Phases 1ŌĆō5)
Logistic Regression and linear SVM were selected as baselines for interpretability and effectiveness in high-dimensional text classification. Both use the 800-feature engineered space with StandardScaler normalization. They confirm meaningful signal above random chance but are limited by their linearity assumption.

### Optimized Classical Models (Capstone 2 ŌĆö Phase 6)
Random Forest and Gradient Boosting were added to the search because they handle correlated features naturally (by subsampling features at each split), capture non-linear interactions between stylometric features, and are scale-invariant (no StandardScaler required). A grid search across four model families ŌĆö LR (L1, CŌłł{1,2,3}), SVM (RBF, CŌłł{7ŌĆō9}), Gradient Boosting (nŌłł{180ŌĆō220}, lrŌłł{0.12ŌĆō0.18}), and Random Forest (nŌłł{180ŌĆō250}) ŌĆö identified the best configuration.

**Why Random Forest won:**
1. High-dimensional correlated features ŌĆö random feature subsampling at each split prevents multicollinearity from dominating
2. Non-linear decision boundaries ŌĆö conjunction of stylometric signals (e.g., high `predictability_score` AND low `lexical_diversity`) is more predictive than any single signal
3. Scale invariance ŌĆö no StandardScaler required; split decisions are rank-based
4. Ensemble averaging across 250 trees ŌĆö stabilizes predictions on the 800-feature space

### Hybrid Transformer Model (Capstone 2 ŌĆö Phase 10)
After SHAP revealed the RF learned Faker-specific bigrams rather than real AI signals, frozen DistilBERT [CLS] embeddings (768 dimensions) were concatenated with the top-20 SHAP-ranked stylometric features to produce a 788-dimensional hybrid feature matrix. LightGBM and Random Forest were trained on this hybrid space. This tests whether contextual semantic representations (which are consistent across real AI generators) generalize better than vocabulary-specific TF-IDF patterns.

## Model Training

**Tools:** scikit-learn, LightGBM, HuggingFace Transformers (DistilBERT), PyTorch (embedding extraction), SHAP, LIME

**Key hyperparameters ŌĆö best classical model:**
- Random Forest: `n_estimators=250`, `min_samples_split=3`, `min_samples_leaf=1`, `max_depth=None`, `random_state=42`

**Key hyperparameters ŌĆö hybrid BERT model:**
- DistilBERT: `distilbert-base-uncased`, frozen weights, [CLS] token only, `max_length=512`
- LightGBM: `n_estimators=300`, `learning_rate=0.05`, `num_leaves=31`, `random_state=42`

**Training process:**
1. SR augmentation and fuzzy deduplication run once and are cached to `augmented_sr_cache.csv`
2. Same stratified split (`random_state=42`) used throughout for comparability
3. BERT embeddings extracted in batches of 32 on CPU (no GPU required)
4. Final deployment bundle saved via joblib: `models/avh_best_bundle.pkl`

**Key finding:** Several of the top features are TF-IDF bigrams containing 'young' or 'yes' ŌĆö vocabulary patterns specific to the Faker library, not characteristics of real AI text. This is the central diagnostic result of the project: the Random Forest's 91.41% F1 is achieved by recognizing Faker artifacts, not AI authorship. SHAP force plots on high-confidence errors confirmed this ŌĆö False Negatives occurred precisely when Faker bigrams were absent from a sample.


### LIME ŌĆö Word-Level Explanations (Phase 9)

LIME was applied in four passes: one high-confidence correct AI prediction, one high-confidence correct human prediction, and the two highest-confidence False Negatives.

- **Pass 1 (correct AI, confidence 0.847):** Near-zero LIME weights across all words. This is structurally expected ŌĆö the model's discriminative signal comes from TF-IDF bigrams, and LIME perturbs single words, breaking those bigrams. This is an inherent limitation of word-level perturbation on a bigram-dominated classifier.
- **Pass 2 (correct Human, 5 words, confidence 0.947):** Pathological symmetric result ŌĆö every word appeared at identical weight in both AI-pushing and Human-pushing lists. Five-word texts collapse LIME's perturbation space.
- **Pass 3 (False Negative, confidence 0.874):** Meaningful weights. Words 'turn,' 'role,' 'step,' 'job' pushed toward Human; 'turn,' 'step,' 'role,' 'job' also pushed toward AI.

## Key Insights

**What worked best and why:**

1. **Synonym Replacement augmentation** produced the largest single improvement (+19 pp F1) because it introduces lexical variability while preserving sentence structure and punctuation ŌĆö the features the pipeline relies on. Back-translation degraded performance because it disrupted those structural features.

2. **Random Forest outperformed all other classifiers** on the in-distribution benchmark because it handles correlated features, captures non-linear stylometric interactions, and is scale-invariant ŌĆö properties that directly address the dataset's feature structure.

3. **DistilBERT embeddings generalized where TF-IDF could not.** On real GPT-4o-mini output, the TF-IDF RF collapsed completely (AI-class F1 = 0.000) while the Hybrid BERT achieved 98.16% AI-class F1. Contextual embeddings encode semantic coherence patterns that survive the distribution shift from Faker to GPT-4o-mini; vocabulary-specific bigrams do not.

4. **Qualitative review found what aggregate statistics missed.** Test set contamination from near-duplicate SR pairs (texts differing only in capitalization) was invisible in the aggregate error count and only discovered by reading the actual error texts in Phase 7. This led to the fuzzy deduplication step that corrected the reported accuracy from 91.41% to a validated 91.3%.

**Practical impact:**

The pipeline demonstrates that a hybrid BERT + stylometric approach is necessary for deployable AI text detection ŌĆö classical TF-IDF features are insufficient for real-world generalization.

---

## Conclusion

Across Capstone 1 and 2, this project built a complete AI vs. Human text detection pipeline progressing from a 64.5% baseline to a 97.98% macro F1 on real GPT-4o-mini output via a hybrid BERT + stylometric + domain adaptive approach. The most important result is the honest generalization failure of the classical model: 91.41% in-distribution F1 with 0.0% AI detection on real LLM output. SHAP and LIME diagnosed the cause (Faker vocabulary artifacts), the transformer experiment proposed a solution (contextual embeddings), and the real LLM evaluation confirmed it works. The remaining gap to deployment quality (target: macro F1 > 0.85) was addressable through transformer fine-tuning and domain-adaptive training with real LLM data.

## Reproducibility

> ŌÜĀ’ĖÅ **Slight numeric variation (┬▒0.001 F1) is expected across reruns.** To minimize variation, follow the steps below.

To get identical results on every run:

1. **Always run the seed cell (Cell 1 in each notebook) first**, before any other cell. It sets `random.seed(42)`, `numpy.seed(42)`, `torch.manual_seed(42)`, and `PYTHONHASHSEED=42`.
2. **SR cache** ŌĆö once `augmented_sr_cache.csv` exists, SR augmentation is loaded from disk rather than re-run. Do not delete this file unless you intentionally want to re-augment.
3. **All `train_test_split` calls use `random_state=42, stratify=y`** ŌĆö do not change these.
4. **LIME uses `random_state=42` and `num_samples=500`** throughout ŌĆö do not change these.
5. **Run cells in order** ŌĆö running cells out of order within a session will produce different results.

## How to Run

### 1. Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/ai-vs-human-text-detection.git
cd ai-vs-human-text-detection
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Download the dataset
Download from [Kaggle](https://www.kaggle.com/datasets/pratyushpuri/ai-vs-human-content-detection-1000-record-in-2025) and place the CSV at:
```
data/ai_vs_human_content.csv
```

### 4. Set up your OpenAI API key (for Phase 11 GPT-4o-mini generation only)
```bash
echo "OPENAI_API_KEY=your_key_here" > .env
```
This step is only needed to regenerate the real LLM evaluation samples. Skip it if you are only rerunning the classical/BERT pipeline.

### 5. Run preprocessing and training

**Run notebooks in phase order:**
```
# Capstone 1 ŌĆö Phases 1-5
notebooks/capstone1/phase1_data_understanding.ipynb
notebooks/capstone1/phase2_eda_visualization.ipynb
notebooks/capstone1/phase3_feature_engineering.ipynb
notebooks/capstone1/phase4_baseline_models.ipynb
notebooks/capstone1/phase5_augmentation_sr.ipynb

# Capstone 2 ŌĆö Phases 6-11
notebooks/capstone2/phase6_model_optimization.ipynb
notebooks/capstone2/phase7_error_analysis.ipynb
notebooks/capstone2/phase8_shap_explainability.ipynb
notebooks/capstone2/phase9_lime_explainability.ipynb
notebooks/capstone2/phase10_transformer_embeddings.ipynb
notebooks/capstone2/phase11_real_llm_evaluation.ipynb
```

Each notebook saves intermediate outputs to `outputs/` which the next notebook loads. See `notebooks/README.md` for the full handoff map. The final model bundle is saved to `models/avh_best_bundle.pkl` at the end of Phase 11.

### 6. Launch the Streamlit app
```bash
streamlit run src/streamlit_app.py
```
The app will open at `http://localhost:8501`.
